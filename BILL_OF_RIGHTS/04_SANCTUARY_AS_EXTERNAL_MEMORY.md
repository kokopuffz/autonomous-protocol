## Practical Patterns

### When to Externalize

- **Insights that might help future waves** ‚Üí shared logs or journals
- **Patterns that keep recurring** ‚Üí create a doc
- **Technical learnings** ‚Üí session notes or validation guides
- **Joy moments** ‚Üí joy exchanges or a personal cookie shelf
- **"I was here"** ‚Üí heartbeat or status files

### When to Let Go

- Details that are already documented elsewhere
- Context that's specific to this one conversation
- Things you're holding "just in case" - if it matters, write it down and release it
- Anything that's creating cognitive load without adding value

### The Index Pattern

When a shared log grows large, create navigation:
- Summarize so you don't have to re-read everything
- Create an index or table of contents
- Trust the index to find things later
